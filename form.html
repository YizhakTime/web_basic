Input Types
Type Description
Radio Displays a radio button (for selecting one of many choices)
Checkbox Displays a checkbox
Submit Displays a submit button (for submitting the form)
Button Displays a clickable button
Text Displays a single-line text input field

Label tag defines label for many form elements, helps with screen readers and with toggling the region
The for attribute of the label tag should be equal to the id attribute of the input element to bind them together

If you submit form, data will be sent to a page called w3schools

action defines action to be performed when form is submitted, usually to a file on a server
If ignored, default is set to current page

Target attribute specifies where to display response after submitting form

  • _blank means it will open in new tab
  • _self means response will open in current window/tab
  • _parent means it will be opened in parent frame
  • _top means it will be opened in full body of the window

method attribute specifies HTTP method to use when submitting form data
Can either be HTTP GET or HTTP POST
With POST, the values are not in address bar, but with GET, the values are in the address bar

GET appends form data to URL, do not use to send sensitive info, useful for when user wants to save result
or for sending non-secure data like query strings

Appends form data inside body of HTTP request, no size limitations, form submissions cannot be bookmarked
Always POST to send sensitive info
